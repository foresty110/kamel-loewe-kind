# 게시판 구현 TODO

`docs/board-*.html` 화면 기획서를 기준으로, 화면별로 구현해야 할 기능을 정리했다.
화면에서 할 일과 백엔드에서 처리해야 할 로직을 나눠서 적었다.

## 공통 기반

- [x] 게시글, 댓글, 첨부파일, 카테고리 테이블을 설계한다. 게시글 하나에는 여러 개의 첨부파일과 여러 개의 댓글이 달릴 수 있다.
- [x] 카테고리 데이터를 미리 등록한다. 윤리학, 철학사, 형이상학, 인식론, 논리학, 정치철학 등이며, 별도 관리 화면 없이 DB에 직접 입력해둔다.
- [ ] 비밀번호는 그대로 저장하지 않는다. 글을 등록할 때 비밀번호를 해시로 변환해서 저장하고, 나중에 확인할 때는 입력받은 값을 같은 방식으로 변환해서 저장된 값과 비교한다.

## 게시판 목록 (`docs/board-list.html`)

**화면**
- [ ] 카테고리를 고르는 드롭다운, 제목·작성자·내용을 검색하는 검색창, 등록일 시작일과 종료일을 고르는 영역을 만든다.
- [ ] 검색된 게시글의 총 건수를 표시한다.
- [ ] 목록 테이블에 첨부파일 아이콘, 제목, 작성자, 조회수, 등록일시와 수정일시를 표시한다. 제목이 80자를 넘으면 뒤를 '…'으로 줄여서 보여주고, 수정한 적이 없는 글은 수정일시 자리에 '-'를 표시한다.
- [ ] 페이지 번호를 눌러 이동하는 페이지네이션을 만든다. 페이지를 이동해도 현재 선택한 카테고리, 검색어, 기간 같은 검색조건이 그대로 유지되어야 한다.

**백엔드**
- [ ] 드롭다운에 넣을 카테고리 전체 목록을 조회하는 쿼리를 만든다.
- [ ] 화면에서 넘어온 카테고리, 검색어, 등록일 시작일, 등록일 종료일, 페이지 번호 값을 받아서 처리한다.
- [ ] 값이 비어 있을 때 기본값을 적용한다. 카테고리를 고르지 않았으면 전체 카테고리로, 등록일 범위를 지정하지 않았으면 최근 1년으로 처리한다.
- [ ] 검색어 하나로 제목, 작성자, 내용 세 항목 중 어디든 일치하면 검색되도록 쿼리를 작성한다.
- [ ] 이 조건에 맞는 게시글 목록을 조회하는 쿼리와, 같은 조건에 맞는 전체 건수를 조회하는 쿼리를 따로 만든다. 건수는 페이지네이션을 계산하는 데 쓴다.
- [ ] 페이지 번호를 이용해서 한 번에 10건씩만 조회되도록 처리한다.
- [ ] 목록의 각 게시글에 첨부파일이 있는지 여부도 함께 조회해서 아이콘 표시에 쓴다.
- [ ] 전체 건수를 기준으로 전체 페이지 수를 계산해서 화면에 전달한다.

## 게시글 보기 (`docs/board-view.html`)

**화면**
- [ ] 제목, 카테고리, 등록일, 작성자, 조회수를 표시한다.
- [ ] 본문 내용을 표시한다.
- [ ] 첨부파일 목록을 파일명과 확장자로 표시하고, 클릭하면 다운로드되는 링크를 만든다.
- [ ] 댓글 목록을 오래된 순서대로 표시해서 최신 댓글이 맨 아래에 오도록 하고, 댓글을 입력하는 창을 함께 둔다. 댓글이 하나도 없으면 입력창만 보이게 한다.
- [ ] 목록으로 가는 버튼, 수정 버튼, 삭제 버튼을 둔다. 삭제 버튼을 누르면 비밀번호 확인 화면이 뜨도록 연결한다.

**백엔드**
- [ ] 게시글 번호로 상세 내용을 조회한다. 존재하지 않는 번호를 요청하면 오류로 처리한다.
- [ ] 상세 내용을 조회할 때마다 조회수를 1 늘리는 처리도 함께 실행한다. 새로고침할 때마다 중복으로 늘어나는 문제는 이번 범위에서는 다루지 않는다.
- [ ] 게시글에 연결된 첨부파일 목록을 조회한다.
- [ ] 첨부파일 다운로드를 처리한다. 요청받은 파일 번호로 서버에 저장된 실제 파일을 찾아서 그 내용을 읽어 원본 파일명으로 응답한다. 파일이 저장된 실제 경로나 주소는 사용자에게 노출하지 않는다.
- [ ] 게시글에 달린 댓글을 등록일이 오래된 순서대로 조회한다.
- [ ] 댓글 등록을 처리한다. 게시글 번호, 작성자 이름, 내용을 받아서 저장하고, 등록일시는 서버에서 현재 시각으로 자동 기록한다.
- [ ] 삭제를 처리한다. 입력받은 비밀번호를 저장된 값과 비교해서 일치하면 게시글을 삭제한다. 일치하지 않으면 실패했다고 응답한다. 댓글과 첨부파일 DB 행은 CASCADE 설정 덕분에 게시글 삭제와 함께 자동으로 지워진다.
- [ ] 서버에 실제로 업로드되어 있는 첨부파일은 DB와 별개라 CASCADE로 지워지지 않는다. 게시글을 지우기 **전에** 첨부파일 목록(저장 경로)을 먼저 조회해두고, 게시글 삭제 후 그 경로들로 실제 파일도 직접 삭제하는 단계를 따로 넣는다.

## 게시글 등록 (`docs/board-write.html`)

**화면**
- [ ] 카테고리 선택, 작성자, 비밀번호, 비밀번호 확인, 제목, 내용을 입력하는 폼과 여러 개의 파일을 첨부할 수 있는 영역을 만든다.
- [ ] 자바스크립트로 입력값을 검증한다. 작성자는 3자 이상 5자 미만, 비밀번호는 4자 이상 16자 미만이며 영문과 숫자와 특수문자를 모두 포함해야 하고 비밀번호 확인 값과 일치해야 한다. 제목은 4자 이상 100자 미만, 내용은 4자 이상 2000자 미만이어야 한다.
- [ ] 검증에 실패하면 어느 필드가 문제인지 화면에 표시한다.
- [ ] 저장에 실패하면 서버 쪽 검증 실패도 포함해서 입력했던 값을 그대로 화면에 남겨둔다. 취소를 누르면 목록으로 이동한다.

**백엔드**
- [ ] 서버에서도 화면과 같은 규칙으로 입력값을 다시 검증한다. 화면을 거치지 않고 직접 요청을 보내서 검증을 건너뛰는 경우를 막기 위해서다.
- [ ] 검증에 실패하면 어떤 항목이 왜 실패했는지 구분해서 알려주는 응답을 만든다.
- [ ] 비밀번호를 해시로 변환한다.
- [ ] 게시글을 저장한다. 카테고리, 작성자, 제목, 내용, 비밀번호를 변환한 값, 등록일시를 게시글 테이블에 새로 저장한다.
- [ ] 업로드된 파일을 서버에 저장하고, 원본 파일명과 저장 위치를 방금 만들어진 게시글 번호와 연결해서 첨부파일 테이블에 기록한다. 파일을 첨부하지 않아도 정상적으로 처리되어야 한다.
- [ ] 저장에 성공하면 새로 만들어진 게시글 번호를 돌려줘서 상세 화면으로 이동할 수 있게 한다.

## 게시글 수정 (`docs/board-edit.html`)

**화면**
- [ ] 카테고리, 등록일, 수정일, 조회수는 수정할 수 없는 읽기 전용으로 보여준다.
- [ ] 작성자, 비밀번호, 제목, 내용을 입력하는 폼에 기존 값을 채워서 보여준다. 비밀번호 칸만 비워둔다.
- [ ] 기존 첨부파일 목록에 다운로드 버튼과 삭제 버튼을 표시하고, 새 파일을 추가로 첨부할 수 있게 한다.
- [ ] 삭제 버튼을 누른 첨부파일은 화면에서만 빠진 것처럼 보이게 하고, 실제로는 저장 버튼을 눌러야 삭제가 반영된다. 취소를 누르면 삭제 표시가 사라지고 원래대로 돌아간다.
- [ ] 클라이언트단 유효성 검증은 등록 화면과 같은 규칙을 적용한다.

**백엔드**
- [ ] 게시글 번호로 기존 값을 조회해서 수정 폼에 채워준다. 카테고리, 작성자, 제목, 내용, 첨부파일 목록, 등록일, 수정일, 조회수를 모두 불러온다.
- [ ] 저장 요청이 오면 비밀번호를 먼저 저장된 값과 비교하고, 일치할 때만 다음 단계로 진행한다.
- [ ] 서버에서도 등록과 같은 규칙으로 입력값을 검증한다.
- [ ] 검증과 비밀번호 확인을 모두 통과하면 제목과 내용을 새 값으로 바꾸고, 수정일시를 현재 시각으로 갱신한다.
- [ ] 삭제하기로 한 첨부파일은 저장을 확정하는 시점에만 실제로 지운다. 저장하기 전에는 지우지 않는다.
- [ ] 새로 첨부된 파일은 등록 때와 같은 방식으로 저장하고 게시글과 연결한다.

## 비밀번호 확인 (`docs/board-password-confirm.html`)

**화면**
- [ ] 게시글 보기 화면에서 삭제 버튼을 누르면 비밀번호를 입력하는 작은 창을 띄운다.
- [ ] 취소 버튼과 확인 버튼을 두고, 비밀번호가 맞지 않으면 알림창으로 알려준다.

**백엔드**
- [ ] 입력받은 비밀번호를 해당 게시글에 저장된 값과 비교하는 확인 로직을 만든다. 이 로직은 등록, 수정, 삭제 여러 화면에서 공통으로 재사용할 수 있게 만든다.
- [ ] 이 로직은 비밀번호가 맞는지만 판단해서 응답하고, 삭제처럼 그다음에 할 일은 이 로직을 불러온 쪽에서 이어서 처리한다.
